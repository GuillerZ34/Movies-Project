# 🎬 MovieLens Word Cloud

This project loads the [MovieLens](https://grouplens.org/datasets/movielens/) ratings dataset into a SQLite database, computes a weighted movie ranking, and renders the top-rated movie titles as an interactive D3.js word cloud — sized by number of votes.

## How it works

```
movielens.csv
      │
      ▼
openmovies.py  ──►  mdb.sqlite (Movies, Genres, Movies_Genres, Ratings)
      │
      ├──► movievis.py    (prints top 10 movies by weighted rating, IMDB-style)
      │
      └──► moviesvis2.py  ──►  gtitle.js (top 100 titles sized by vote count)
                                    │
                                    ▼
                              gtitle.htm (D3.js word cloud, opens in browser)
```

## Project structure

```
movielens-wordcloud/
├── movielens.csv          # Raw dataset (MovieLens ratings export)
├── openmovies.py          # Parses the CSV and builds the SQLite database
├── movievis.py            # Prints the top 10 movies by weighted rating
├── moviesvis2.py          # Generates gtitle.js from vote counts
├── gtitle.js              # Word cloud data (generated by moviesvis2.py)
├── gtitle.htm              # Word cloud visualization (open in a browser)
├── d3.v2.js                # Third-party library: D3.js v2.8.1
└── d3.layout.cloud.js      # Third-party library: D3 word cloud layout
```

## Requirements

- Python 3 (only standard library modules are used: `sqlite3`, `re`, `datetime`, `json` — no `pip install` needed)
- A modern web browser to open `gtitle.htm`

## Usage

Run the scripts in this order from the project folder:

### 1. Build the database

```bash
python openmovies.py
```

This reads `movielens.csv` and creates `mdb.sqlite` with four tables: `Movies`, `Genres`, `Movies_Genres`, and `Ratings`. Re-running this script drops and rebuilds all tables from scratch.

### 2. (Optional) Print the top 10 movies by weighted rating

```bash
python movievis.py
```

Uses a weighted rating formula (similar to IMDB's approach) that balances a movie's average rating against how many votes it received, so movies with very few — but very high — ratings don't unfairly outrank well-established favorites.

### 3. Generate the word cloud data

```bash
python moviesvis2.py
```

This finds the 100 most-voted movies and writes `gtitle.js`, where each title's font size is scaled between 20 and 80 based on its relative number of votes.

### 4. View the word cloud

Open `gtitle.htm` directly in a web browser. It loads `d3.v2.js`, `d3.layout.cloud.js`, and the generated `gtitle.js` to render the titles as a word cloud — larger text means more ratings.

## Notes

- `mdb.sqlite` is git-ignored since it's a generated file — anyone can recreate it by running `openmovies.py` against `movielens.csv`.
- `d3.v2.js` and `d3.layout.cloud.js` are third-party libraries included as-is; see [LICENSE](LICENSE) for their attribution.
